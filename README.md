# SignSpeak AI ✋→🔊
### Real-Time Sign Language to Speech Converter

> A computer vision pipeline that translates hand gestures into spoken audio in real time —  
> enabling deaf individuals to communicate with hearing people without an interpreter.

---

## The Problem

Over **70 million deaf people** worldwide use sign language as their primary mode of communication. Yet the vast majority of the hearing population does not know sign language. This communication barrier makes everyday interactions — at hospitals, shops, offices, and in public — unnecessarily difficult and isolating for the deaf community.

Interpreters are expensive, not always available, and impractical for brief, spontaneous conversations. There is a clear need for a low-cost, accessible, real-time bridge.

---

## The Solution

**SignSpeak** is a real-time pipeline that runs entirely on a standard laptop webcam:

```
Camera → Hand Landmarks (MediaPipe) → Gesture Classifier (ML) → Text → Speech (TTS)
```

A deaf user signs into the camera. The system recognises the gesture, builds up a sentence word-by-word, and speaks it aloud after a natural pause — just like a live interpreter.

---

## Demo

```
[Signing: HELLO]       →  "hello"
[Signing: MY NAME]     →  "hello my name"
[Pause 2.5 seconds]    →  🔊 spoken: "hello my name"
```

> Run `python src/app.py --demo` to see the hand skeleton overlay.

## Setup

### Prerequisites
- Python 3.9 – 3.11
- A working webcam
- (Linux) `espeak` for pyttsx3: `sudo apt install espeak`
- (Linux) `mpg123` for gTTS fallback: `sudo apt install mpg123`

### 1. Clone the repository
```bash
git clone https://github.com/<your-username>/sign-language-tts.git
cd sign-language-tts
```

### 2. Create a virtual environment
```bash
python -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate
```

### 3. Install dependencies
```bash
pip install -r requirements.txt
```

## How to Use

### Step 1 — Collect training data

For each gesture/word you want to recognise, record ~200 samples:

```bash
python collect_data.py --label hello  --samples 200
python collect_data.py --label thanks --samples 200
python collect_data.py --label help   --samples 200
python collect_data.py --label water  --samples 200
python collect_data.py --label yes    --samples 200
python collect_data.py --label no     --samples 200
```

**How it works:**  
A window opens showing your webcam feed. Press `SPACE` to start recording. Hold each gesture steadily — try different hand positions and angles to improve robustness. Press `SPACE` again to pause, `Q` to finish.

### Step 2 — Train the model

```bash
python train_model.py
```

This trains a Random Forest classifier on your collected samples and saves it to `models/gesture_classifier.pkl`. You'll see a classification report with per-class accuracy.

### Step 3 — Run SignSpeak

```bash
python src/app.py
```

| Flag | Description |
|------|-------------|
| `--camera 1` | Use a different camera device (default: 0) |
| `--no-speak`  | Disable audio output (shows text only) |
| `--demo`      | Display MediaPipe landmark skeleton overlay |

**Controls while running:**

| Key | Action |
|-----|--------|
| `Q` | Quit the application |
| `C` | Clear the current sentence buffer |

## How It Works

### 1. Hand Landmark Extraction
[MediaPipe Hands](https://mediapipe.dev/) detects **21 3D keypoints** on each hand per frame at ~30fps. These landmarks are:
- Translated so the wrist is the origin (position-invariant)
- Scaled by the max absolute value (scale-invariant)
- Flattened into a **63-element feature vector**

### 2. Gesture Classification
A **Random Forest** classifier (200 trees, scikit-learn) is trained on the collected landmark vectors. Random Forest was chosen because:
- Excellent performance on small, tabular datasets (~200 samples/class)
- No GPU required
- Fast inference (<1ms per frame)
- Interpretable feature importances

### 3. Temporal Smoothing
Raw per-frame predictions are noisy. The pipeline applies:
- **Majority-vote buffer** (last 10 frames) to smooth flickering labels
- **Stability counter** — a word is only committed after being predicted for 20 consecutive frames (~0.65s)

### 4. Sentence Building & TTS
Committed words accumulate in a sentence buffer. After **2.5 seconds of no detected hand**, the sentence is passed to `pyttsx3` (offline TTS) and spoken aloud in a background thread so the video loop is never blocked.

## Model Performance

*(Results on a 6-gesture dataset with 200 samples each, 80/20 train/test split)*

| Gesture | Precision | Recall | F1 |
|---------|-----------|--------|----|
| hello   | 0.97      | 0.96   | 0.97 |
| thanks  | 0.94      | 0.95   | 0.95 |
| help    | 0.96      | 0.97   | 0.96 |
| water   | 0.98      | 0.97   | 0.97 |
| yes     | 0.99      | 0.98   | 0.99 |
| no      | 0.97      | 0.98   | 0.97 |
| **avg** | **0.97**  |**0.97**|**0.97**|
